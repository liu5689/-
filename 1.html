小白Hexo搭建个人博客网站（2019版）

一.安装Node.js

​ 官网下载最新版，安装选项全部默认，一路点击Next。

最后安装好之后，按Win+R打开运行提示框，输入cmd打开命令提示符，输入 node -v 和 npm -v，如果出现版本号就安装成功了。

二.安装Git

​ 官网下载最新版，安装选项全部默认，一路点击Next（其中有一个选项选Add to path).

三.注册Github账号

，
新建一个仓库，项目名字后面一定要加.github.io后缀，README初始化也要勾上。名称一定要和你的github名字完全一样，比如你github名字叫abc，那么仓库名字一定要是abc.github.io。项目建成后，点击Settings，向下拉到最后有个GitHub Pages，点击Choose a theme选择一个主题。然后等一会儿，再回到GitHub Pages，点击那个链接，就会出现自己的网页啦！

四.安装Hexo

​ 在合适的地方新建一个文件夹，用来存放自己的博客文件，比如我的博客文件都存放在E:\Hexo\Myblog目录下。

在该目录下右键点击Git Bash Here，打开git的控制台窗口，以后我们所有的操作都在git控制台进行，就不要用Windows自带的控制台了。
定位到该目录下，输入$ npm i hexo-cli -g 安装Hexo。会有几个报错，无视它就行。

安装完后输入$ hexo -v 验证是否安装成功。

然后就要初始化我们的网站，,在该 Myblog 文件夹上右键鼠标，点击 Git Bash Here，输入以下命令，

hexo init 初始化本地网站，显示 【 INFO Start blogging with Hexo!】则安装成功。 接着输入 npm install 安装必备的组件。这样本地的网站配置也弄好啦，
git bash here控制台输入$ hexo g 生成静态网页，然后{输入$ hexo s 打开本地服务器}, 显示以下信息说明操作成功：
【INFO Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.】,

复制GIT控制台上的网址，然后在浏览器上粘贴打开http://localhost:4000/网页，就可以看到我们的博客啦！（本地服务器一点要在打开状态，否则打不开网页） 按ctrl+c关闭本地服务器。【本地可看到博客，外网不能】

五.连接Github与本地

首先在Myblog文件夹上右键打开git bash here，然后输入下面命令：
git config –global user.name “此处填你的github用户名”
git config –global user.email “此处填你的邮箱”

然后生成密钥SSH key:输入命令
$ ssh-keygen -t rsa -C “your email@example.com“ //引号里面填写你的邮箱地址，比如我的是liu.guanbing@163.com，之后会出现：

【Generating public/private rsa key pair.
INFO Start blogging with Hexo!

Enter file in which to save the key (/c/Users/you/.ssh/id_rsa): 】//到这里可以直接回车，将密钥按默认文件进行存储，这时会显示y/n，输入y，然后会出现：
【Enter passphrase (empty for no passphrase): 】//这里是要你输入密码，其实不需要输什么密码，直接回车就行
【Enter same passphrase again:】，直接回车接下来屏幕会显示
【Your identification has been saved in /c/Users/you/.ssh/id_rsa.
Your public key has been saved in /c/Users/you/.ssh/id_rsa.pub.
The key fingerprint is:
这里是各种字母数字组成的字符串，结尾是(你的邮箱 )
The key’s randomart image is:
这里也是各种字母数字符号组成的字符串(在虚线方框中）】
接着 复制代码 ： 运行以下命令， clip < ~/.ssh/id_rsa.pub, 将公钥的内容复制到系统粘贴板上，也可以用记事本打开 id_rsa.pub 文件（一般在 C:\Users\用户名.ssh 目录下）

​ 在 GitHub 账户中添加你的公钥
​ ① 登陆 GitHub，在右上角的倒三角中进入 Settings： ② 点击 SSH and GPG Keys： ③ 选择 New SSH key： ④ 粘贴密钥：

六.将本地的Hexo文件更新到Github的库中

① 登录 Github 打开自己的项目 your name.github.io
② 鼠标移到 Clone or download 按钮，选择 Use SSH （Use HTTPS 也行，不过容易出现连接不上的问题）
③ 一键复制地址（按长条框后的图标）
④ 打开你创建的 Myblog 文件夹，右键用记事本（Notepad++）打开该文件夹下的 _config.yml 文件 ，这是博客的配置文件，在这里你可以修改与博客相关的各种信息。
⑤ 按下图修改 _config.yml 文件并保存
修改最后一行的配置：(每个冒号后面都有一个空格）

deploy: //找到此关键字
type: git //type后面写上git
repository: //你自己的github项目地址（,即上面3中复制的地址）
branch: master//没有改动分支
⑥ 在 Myblog 文件夹下执行以下命令 npm install hexo-deployer-git –save(安装部署插件，很重要，否则hexo与github不能同步更新）；
​ hexo d (部署到github)
⑦ 访问博客
你的博客地址：https://你的用户名.github.io，比如我的是：https://liu5689.github.io，现在每个人都可以通过此链接访问你的博客了。

七.在博客上发表文章

​ 博客已经成功搭建了，但是我们该怎么写博客呢？
​ ① 新建一个空文章，输入以下命令，会在项目 \Hexo\source_posts 中生成 文章标题.md 文件，文章标题根据需要命名 。 $ hexo n “文章标题

② 用编辑器编写文章
推荐 Windows使用Typory。文章标题，标签，分类，封面图片，摘要等，可以在 Front-matter 里面配置（Front-matter 是文件最上方以 — 分隔的区域，用于指定个别文件的变量，官方文档：https://hexo.io/zh-cn/docs/front-matter ）。 当我们用编辑器写好文章后，先输入 hexo g 生成静态网页，再输入 hexo s 启动本地服务器在浏览器预览，可以就使用以下命令将其推送到服务器上 $ hexo d ，现在访问你的博客就可以看见写好的文章啦！

【typora + hexo博客中插入图片的技巧】 事先声明，所有博客文件均保存在 hexo/_posts/文件夹下

首先在 hexo > source目录下建一个文件夹叫images，用来保存博客中的图片。
然后打开Typora的 文件 > 偏好设置，进行如下设置：在面板上选择“图像”–》“复制到指定路径”–》选择文件夹（../../../source/images)–》后面的选项都勾选上，这样的话所有的博客中的图片都将会保存到 /source/images/该博客md文件名/图片名称

但是仅仅这样设置还不够，这样设置在typora中倒是能看图片了，但是使用的却是相对于当前md文件的相对路径，可是如果启动hexo，是要用服务器访问的，而服务器显然无法根据这个相对路径正确访问到图片，因此还需要在typora中进行进一步设置。

在typora菜单栏点击 格式->图像->设置图片根目录，将hexo/source作为其根目录即可。

一定要先设置了图片根目录后再插入图片，否则图片路径会不正确喔！

八.为博客更换自己喜欢的主题

​ 博客也搭建好了，文章也会写了，但是！！！默认的主题并不喜欢怎么办？现在，我们就来为自己的博客更换自己喜欢的主题。点击此处进入 Hexo 官网的主题专栏，我们可以看见有许多的主题供我们选择,我们要做的就是把主题克隆过来，在此我们以主题 Matery 为例，点击图片下的题目 ，页面就会跳转到该主题所有的相关文件在 Github 上的地址，复制该地址，再打开 Myblog文件夹下的 themes 目录，右键 Git Bash Here，输入以下命令：
​ $ git clone 此处填写你刚才复制的主题地址

等待下载完成后即可在 themes 目录下生成 hexo-theme-(该主题名称） 文件夹，然后打开 Myblog 文件夹下的配置文件 _config.yml ，找到关键字 theme，修改参数为：theme：hexo-theme-(该主题名称）（其他主题修改成相应名称即可），再次注意冒号后面有一个空格！
返回 Myblog 目录，右键 Git Bash Here ，输入以下命令开始部署主题：
hexo g ； hexo s
此时打开浏览器，访问 http://localhost:4000/ 就可看见我们的主题已经更换了，如果感觉效果满意，我们就可以把它部署到Github上了。

​ I. 打开 Myblog文件夹，右键 Git Bash Here ，输入以下命令：
​ hexo clean //该命令的作用是清除缓存，若不输入此命令，服务器有可能更新不了主题
​ hexo g -d 重要提醒：【 每次优化主题后，都有重复这两步，否则就不能成功部署到博客上。】

此时访问自己的博客即可看见更换后的主题，但我们仍然需要对主题的相关配置进行修改，比如网站标题，图标等等，Myblog 中有两份主要的配置文件，名称都是 _config.yml ，它们均是用于站点配置使用的。其中，一份位于站点根目录下，主要包含Myblog 本身整站的配置；另一份位于主题hexo-theme-(该主题名称）目录下，这份配置由主题作者提供，主要用于配置主题相关的选项，一般 _config.yml 文件里都有相关注释，主题也有相关文档，按需修改即可。
